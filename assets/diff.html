Code:

$git diff /*show diff of unstaged changes*/

Explanation:

Without any extra arguments, a simple git diff will display in unified diff format (a patch) what code or content you've changed in your project since the last commit that are not yet staged for the next commit snapshot.

Code:

$git diff --cached /* show diff of staged changes */

Explanation:

The git diff --cached command will show you what contents have been staged. That is, this will show you the changes that will currently go into the next commit snapshot

Code:

$git diff HEAD /* show diff of all staged or unstaged changes */

Explanation:

If you want to see both staged and unstaged changes together, you can run git diff HEAD - this basically means you want to see the difference between your working directory and the last commit, ignoring the staging area

Code:

$git diff --stat /* show summary of changes instead of a full diff */

Explanation:

If we don't want the full diff output, but we want more than the git status output, we can use the --stat option, which will give us a summary of changes instead
